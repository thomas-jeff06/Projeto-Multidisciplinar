# Projeto-Multidisciplinar

# Sistema Web de Pedidos para Lanchonete

## Sobre o Projeto

Este projeto consiste no desenvolvimento de uma aplicação web responsiva voltada para gerenciamento de pedidos em uma lanchonete, permitindo que clientes realizem pedidos de forma digital, acompanhem o status em tempo real e utilizem funcionalidades de fidelização e promoções.

A aplicação foi projetada seguindo conceitos modernos de desenvolvimento Front-End, priorizando experiência do usuário, organização de fluxos operacionais e integração com serviços externos simulados.

---

# Objetivos do Sistema

* Digitalizar o processo de pedidos;
* Melhorar a experiência do cliente;
* Centralizar o gerenciamento operacional;
* Simular integração com sistemas externos de pagamento;
* Aplicar conceitos de responsividade, UX/UI e arquitetura de sistemas.

---

# Funcionalidades Principais

## Cliente

* Cadastro e autenticação de usuários;
* Visualização dinâmica do cardápio por unidade;
* Adição e remoção de itens do carrinho;
* Realização de pedidos;
* Acompanhamento do status do pedido;
* Sistema de fidelização;
* Visualização de promoções e campanhas;
* Simulação de pagamento externo.

## Administração

* Gerenciamento de produtos;
* Gerenciamento de promoções;
* Controle de pedidos;
* Atualização de status dos pedidos;
* Visualização operacional do sistema.

---

# Arquitetura da Solução

O sistema foi estruturado utilizando separação entre:

* Interface Front-End;
* Fluxos de negócio;
* Simulação de integração externa;
* Gerenciamento de estados da aplicação.

A comunicação entre módulos ocorre através de componentes organizados e gerenciamento de estados locais, simulando a estrutura de aplicações modernas integradas a APIs REST.

# Fluxo do Sistema

1. Usuário realiza autenticação;
2. Sistema carrega o cardápio da unidade;
3. Cliente seleciona produtos;
4. Pedido é enviado ao sistema;
5. Fluxo de pagamento externo é simulado;
6. Sistema atualiza o status do pedido;
7. Cozinha recebe o pedido;
8. Cliente acompanha o progresso em tempo real.

---

# Integrações Simuladas

O projeto contempla integração conceitual com:

* Gateway de pagamento;
* Sistema de pedidos;
* Sistema de fidelização.

O processamento financeiro foi representado apenas visualmente, sem utilização de transações reais.
